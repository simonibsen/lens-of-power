# Evidence

> Role: This directory stores **evidence** — concrete facts, data points,
> documented cases, quotes, and observations that ground the framework in
> reality. Evidence is not interpretation — it is raw material that
> supports, challenges, or illustrates the framework's axioms and patterns.
> Each entry is tagged with the axioms and patterns it relates to and with
> its relationship to them (supports / challenges / complicates). Evidence
> that challenges an axiom is the most valuable kind — it tests whether
> the framework's assumptions hold.

## Entry format

Each evidence entry is a markdown file using this structure:

```markdown
# [Short descriptive title]

DATE RECORDED: [YYYY-MM-DD]
SOURCE: [where this fact/data comes from]
SOURCE TYPE: statistic / case study / quote / document / event / data

## Tags
AXIOMS: [comma-separated axiom numbers this relates to, e.g., 3, 5, 7]
PATTERNS: [comma-separated pattern names, if any]
LAYERS: [taxonomy layers involved]
RELATIONSHIP: supports / challenges / complicates

## Content
[The actual evidence — fact, quote, data point, case description]

## Significance
[Brief note on why this matters to the framework — 1-3 sentences]
```

## Naming convention

Files are named: `YYYY-MM-DD-short-description.md`

Example: `2026-03-01-wealth-concentration-statistics.md`

## Usage

When performing analysis, evidence entries can be cross-referenced to
strengthen arguments or challenge assumptions. When reviewing axioms,
evidence entries tagged with that axiom provide the empirical basis for
keeping, refining, or revising it.
